设置 Git 代理主要有两种方法：**通过 `git config` 命令**或**通过设置环境变量**，具体取决于您的需求和代理服务器的类型。

这里为您介绍最常用且推荐的 `git config` 方法（全局设置），请根据您的代理类型和端口进行替换。

------



## 💻 Git 代理设置（`git config` 方法）



使用 `git config --global` 设置的代理是全局的，会应用于您系统上所有 Git 仓库的 HTTP/HTTPS 连接。



### 1. 设置 HTTP 和 HTTPS 代理



您需要为 `http.proxy` 和 `https.proxy` 分别设置代理。请将 `127.0.0.1:1080` 替换为您实际的代理地址和端口。



#### 🔹 针对 SOCKS5 代理 (推荐用于 Shadowsocks/Clash 等)



如果您的代理服务器使用 SOCKS5 协议（例如本地端口是 `7897`）：

Bash

```
git config --global http.proxy 'socks5://127.0.0.1:7897'
git config --global https.proxy 'socks5://127.0.0.1:7897'
```



#### 🔹 针对 HTTP/HTTPS 代理



如果您的代理服务器使用标准的 HTTP 协议：

Bash

```
git config --global http.proxy 'http://127.0.0.1:7897'
git config --global https.proxy 'http://127.0.0.1:7897'
```

------



### 2. 仅为特定域名设置代理 (推荐)



如果您只想在访问 **GitHub** 等特定网站时使用代理，而访问国内代码库时不使用，可以使用以下命令（以 GitHub 为例）：

Bash

```
# 仅对访问 github.com 的 HTTPS 连接使用 SOCKS5 代理
git config --global http.https://github.com.proxy socks5://127.0.0.1:1080
```

------



### 3. 查看当前代理设置



您可以查看 Git 的全局配置列表来确认代理是否设置成功：

Bash

```
git config --global -l | grep proxy
```

------



### 4. 取消代理设置



当您不再需要代理时，可以通过 `unset` 命令取消配置：



#### 🔹 取消所有全局代理



Bash

```
git config --global --unset http.proxy
git config --global --unset https.proxy
```



#### 🔹 取消特定域名的代理 (例如 GitHub)



Bash

```
git config --global --unset http.https://github.com.proxy
```
